# of Unique Words Used Within Artist's First 35,000 Lyrics




This data science project is inspired by Matt Daniels' awesome post. My goal was to replicate his findings with Spanish-speaking rappers. The first step of the project was to create a corpus of lyrics from a number of preselected artists. For this purpose, TextFileGenerator.java and WebCrawler.java scrape the site https://www.musica.com for an artist's lyrics and store them in a .txt file. TextFileGenerator makes use of Jsoup to parse and remove unwanted HTML. The module remove_lang.py polishes our data with the use of the langdetect library by removing lyrics in foreign languages such as English or Portuguese, which would artificially inflate an artist's word count. The file is then processed with python's NLTK library. The module analyse.py creates Text objects for each file and performs a simple tokenizing process to retrieve the unique word counts for each artist, which are then sorted in ascending order. Both .py files require python 3. The results are then saved to results.csv and plotted with the C3 JavaScript library using code from the following repository as a template: https://github.com/huyle333/graphs-from-csv.
